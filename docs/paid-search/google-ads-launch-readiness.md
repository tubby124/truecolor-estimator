### Cumulative-spend hard stop

The one-shot monitor at `scripts/google-ads/hard-stop-monitor.mjs` is allowlisted to True Color customer `1072816342` and only campaign IDs `24048123058`, `24048123061`, and `24048123064`. It verifies the account is CAD with timezone `America/Regina`, reads cumulative cost for the full approved window, validates all three campaign identities, and emits one structured JSON result. It never prints credential values.

Dry-run is the default and cannot mutate Google Ads:

```sh
npm run monitor:google-ads-spend -- --profile=public-pilot
npm run monitor:google-ads-spend -- \
  --profile=controlled-test \
  --window-start=2026-07-20T08:00 \
  --window-end=2026-07-23T08:00
```

The controlled-test window is mandatory, interpreted in `America/Regina`, restricted to whole-hour boundaries, and cannot exceed 72 hours. Hour-segmented Google Ads cost is filtered to `window-start` inclusive and `window-end` exclusive. Its protective threshold is CA$25 against the separately approved CA$30 cap. The fixed public-pilot window is July 20 through August 18 local, with a CA$1,400 protective threshold against the CA$1,500 planning cap. At or above a threshold—or after the selected window ends—the dry-run result is `STOP_REQUIRED` / `WOULD_PAUSE`.

Only the explicit `--execute` flag authorizes a pause:

```sh
npm run monitor:google-ads-spend -- --profile=public-pilot --execute
```

Execute mode pauses all three allowlisted campaigns and verifies paused state by readback. It is idempotent when they are already paused. If cumulative spend cannot be read after account/campaign validation, execute mode attempts the same fail-closed pause and still exits nonzero; if pause or readback cannot be verified, it exits nonzero with `ERROR_PAUSE_UNVERIFIED`. Authentication, account, campaign, or read failures also exit nonzero. Do not treat process execution alone as success—automation must inspect both the exit code and JSON `outcome` / `pauseVerified` fields.

This repository provides the one-shot control only. A production scheduler polling at least every 15 minutes, durable operator/Telegram alert delivery, health monitoring, and the documented manual CA$1,350 fallback remain external launch blockers until configured and proved. Do not run `--execute`, schedule the monitor, or enable campaigns until the corresponding controlled-test or public-pilot approval is recorded.
